**Equal Split:**
- All participants shown with equal amounts
- Auto-calculated: Total ÷ Number of people
- Green checkmarks for fair distribution

**Custom Split:**
- Each participant with individual amount fields
- Running total shown
- Validation to ensure total matches expense amount
- Red indicators if amounts don't add up

**Ratio Split:**
- Percentage or ratio inputs for each participant
- Real-time amount calculation
- Visual bar showing distribution

## 🎨 UI/UX Design Elements

### Color Scheme
- **Primary**: Blue (#4F8CFF) - actions, links, highlights
- **Success**: Green (#22C55E) - completed, positive actions
- **Warning**: Yellow/Orange (#F59E0B) - pending, attention needed
- **Error**: Red (#EF4444) - errors, deletions
- **Neutral**: Gray tones - text, backgrounds, borders

### Typography
- **Headings**: Bold, large size for hierarchy
- **Body**: Regular weight for readability
- **Captions**: Small size for secondary information
- **Buttons**: Medium weight, clear contrast

### Iconography
- **Consistent style**: Line icons throughout
- **Meaningful symbols**: Intuitive representations
- **Size hierarchy**: Important actions larger
- **Color coding**: Status indicated by color

### Layout Patterns
- **Cards**: Information grouped in containers
- **Lists**: Scannable vertical arrangements
- **Grids**: Organized feature displays
- **Forms**: Logical input groupings

## 📱 Mobile-Specific Interactions

### Gesture Support
- **Swipe right**: Edit item
- **Swipe left**: Delete/archive
- **Pull down**: Refresh content
- **Long press**: Context menu
- **Tap**: Primary action

### Navigation Patterns
- **Bottom tabs**: Main app sections
- **Header buttons**: Screen-specific actions
- **Back navigation**: Consistent across screens
- **Modal flows**: Focused tasks

### Responsive Design
- **Adaptive layouts**: Work on all screen sizes
- **Touch targets**: Minimum 44px for accessibility
- **Readable text**: Scales appropriately
- **Accessible colors**: Sufficient contrast ratios
